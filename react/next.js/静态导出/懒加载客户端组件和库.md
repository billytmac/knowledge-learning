 # next/dynamic is a composite of React.lazy() and Suspense. It behaves the same way in the app and pages directories to allow for incremental migration.
 # 场景：用于弹窗组件、不是首屏显示的组件
 # ssr字段举例：如：
 ```
   const LotterySection = dynamic(
    () => import('components/LotterySection'),
    { ssr: true }
    )
    const SignInRulePopup = dynamic(() => import('components/popups/SignInRulePopup'), { ssr: false })

```
- `LotterySection` 用 `ssr: true`  
因为它是页面正文的一部分（即使在下方），通常希望服务端先产出 HTML，保证内容可预渲染、首屏结构完整、SEO/可访问性更稳。  
同时它是懒加载，不会一开始就下载这块 JS，兼顾性能。

- 一堆 `Popup` 用 `ssr: false`  
这些是“用户触发才出现”的交互层，不需要首屏 HTML。设为 `false` 的收益是：  
1. 首屏少渲染一批无用 UI  
2. 避免弹窗库常见的 SSR 问题（依赖 `window/document`、portal 到 `body`、hydration 不一致）  
3. JS 延后到真正打开弹窗时再加载

一句话总结：  
`ssr: true` 给“页面内容型组件”，`ssr: false` 给“触发型、浏览器强依赖、非首屏”的弹层组件。
