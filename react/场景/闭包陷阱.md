## React 闭包陷阱的解决方案

---

### 什么是闭包陷阱

```ts
const [count, setCount] = useState(0)

useEffect(() => {
  const timer = setInterval(() => {
    console.log(count) // 永远是 0，拿到的是旧值
  }, 1000)
  return () => clearInterval(timer)
}, []) // 依赖数组为空，闭包捕获了初始值
```

---

### 方案一：正确设置依赖数组

```ts
useEffect(() => {
  const timer = setInterval(() => {
    console.log(count)
  }, 1000)
  return () => clearInterval(timer)
}, [count]) // 依赖 count，count 变化重新创建
```

缺点：每次 count 变化都会重新创建定时器。

---

### 方案二：使用函数式更新

```ts
// 不读取 state，直接用回调拿最新值
setCount(prev => prev + 1) // prev 永远是最新值
```

适合只需要**基于上一个值更新**的场景。

---

### 方案三：useRef 保存最新值

```ts
const countRef = useRef(count)

useEffect(() => {
  countRef.current = count // 每次同步最新值到 ref
}, [count])

useEffect(() => {
  const timer = setInterval(() => {
    console.log(countRef.current) // 通过 ref 拿最新值
  }, 1000)
  return () => clearInterval(timer)
}, []) // 依赖数组可以为空
```

---

### 方案四：useCallback + 依赖数组

```ts
const handleClick = useCallback(() => {
  console.log(count)
}, [count]) // count 变化时重新生成函数
```

---

### 方案五：自定义 useLatest Hook

把方案三封装成通用 hook：

```ts
function useLatest<T>(value: T) {
  const ref = useRef(value)
  useEffect(() => {
    ref.current = value
  })
  return ref
}

// 使用
const latestCount = useLatest(count)

useEffect(() => {
  const timer = setInterval(() => {
    console.log(latestCount.current) // 永远是最新值
  }, 1000)
  return () => clearInterval(timer)
}, [])
```

---

### 总结

| 方案 | 适用场景 |
|------|---------|
| 正确设置依赖 | 简单场景，依赖变化可以接受重新执行 |
| 函数式更新 | 只需基于上一个值计算新值 |
| useRef | 需要在长生命周期回调里读最新值 |
| useCallback | 事件处理函数依赖 state |
| useLatest | 通用封装，推荐复杂项目使用 |
