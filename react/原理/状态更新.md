React 的不可变更新就是：**不在原对象/数组上直接改**，而是创建一个新对象/新数组再 `setState`。

示例：

```ts
// 不可变更新（正确）
setState(prev => ({ ...prev, count: prev.count + 1 }))
setList(prev => [...prev, newItem])
```

反例：

```ts
// 可变更新（不推荐）
state.count += 1
list.push(newItem)
```

为什么要这样做：React 主要靠“引用是否变化”判断要不要更新。  
你给新引用，React 才能稳定感知变化并触发渲染/effect。

可以这么理解，但更准确是：React 做的是**引用比较（`Object.is`）**，不是深比较“值多少钱/值内容”。

- 基本类型（数字、字符串）比较值本身
- 对象/数组比较引用地址

所以你常听到“浅比较”，本质就是这个。

# setCount(prev => prev + 1)和setCount(count + 1)有什么区别

区别在“是否基于最新值”：

- `setCount(count + 1)`  
用的是当前闭包里的 `count`。连续多次调用时可能丢更新。

- `setCount(prev => prev + 1)`  
React 会把“最新状态”传给 `prev`，连续调用也能正确累加。

所以凡是“新值依赖旧值”，优先用函数式写法 `prev => ...`。

- 对象也是以此类推
