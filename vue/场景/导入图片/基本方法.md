Vite 中导入图片主要有以下几种方式：

**1. 静态导入（最常用）**
```js
import imgUrl from './assets/logo.png'
// imgUrl 是处理后的 URL 字符串
<img src={imgUrl} />
```

**2. 动态导入**
```js
const imgUrl = new URL('./assets/logo.png', import.meta.url).href
// 利用原生 ESM，支持动态路径拼接，SSR 友好
```
new URL(url)
new URL(url, base)
如果 url 是一个相对路径，则必须提供 base，如果 url 是一个绝对路径，则 base 会被忽


**3. `public` 目录直接引用**
```html
<!-- 放在 /public/logo.png，直接用绝对路径 -->
<img src="/logo.png" />
```
public 目录的文件不会被 Vite 处理，原样输出，如果图片变化，需要自己加时间戳


**4.import.meta.glob**
预加载 assets/images 下所有 png
```vue
<script setup>
import { computed, ref } from 'vue'


const imageModules = import.meta.glob('../assets/images/*.png', {
  eager: true,
  import: 'default',
})

const imageName = ref('voucher') // 这里就是动态的 xxx

const dynamicImg = computed(() => {
  return imageModules[`../assets/images/${imageName.value}.png`] || ''
})
</script>

<template>
  <img :src="dynamicImg" class="w-126 h-100 mt-20" />
</template>
```
懒加载 assets/images 下所有 png
```vue
<script setup>
import { ref, watch } from 'vue'

const imageName = ref('voucher') // 动态 xxx
const dynamicImg = ref('')

const imageModules = import.meta.glob('../assets/images/*.png', {
  import: 'default',
})

watch(imageName, async (name) => {
  const key = `../assets/images/${name}.png`
  const loader = imageModules[key]
  dynamicImg.value = loader ? await loader() : ''
}, { immediate: true })
</script>

<template>
  <img :src="dynamicImg" class="w-126 h-100 mt-20" />
</template>

```

**错误写法**
```html
<img :src="`@/assets/images/home/${name}.png`" />
<!-- :src得到的是运行时字符串，@ 别名不会在运行时字符串里被解析。 -->
<img :src="'../assets/images/${name}.png'" />
<!-- :src得到的是运行时字符，Vite 不会把它当成模块资源处理，所以生产环境容易 404（尤其有 base 路径时） -->
 ```
```js
 require(`@/assets/images/home/${name.value}.png`))
报错：Uncaught ReferenceError: require is not defined
因为 Vite 是基于 ES Modules (ESM) 构建的，而 require 是 CommonJS 模块系统的一部分, 属于vue2写法
```
