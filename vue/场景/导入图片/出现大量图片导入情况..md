可以这样写，分两种。

## 1) 放 `public` 目录
目录示例：
`public/images/voucher.png`

在 Vue 里直接拼：

```vue
<script setup>
import { computed, ref } from 'vue'

const imageName = ref('voucher') // 动态 xxx
const dynamicImg = computed(() => `/images/${imageName.value}.png`)
</script>

<template>
  <img :src="dynamicImg" class="w-126 h-100 mt-20" />
</template>
```

如果你的项目有 `base` 路径，建议用：

```ts
const dynamicImg = computed(() => `${import.meta.env.BASE_URL}images/${imageName.value}.png`)
```

## 2) 放 CDN
比如 CDN 根地址是 `https://cdn.xxx.com/activity-images`：

```vue
<script setup>
import { computed, ref } from 'vue'

const imageName = ref('voucher')
const CDN_BASE = 'https://cdn.xxx.com/activity-images'
const dynamicImg = computed(() => `${CDN_BASE}/${imageName.value}.png`)
</script>

<template>
  <img :src="dynamicImg" class="w-126 h-100 mt-20" />
</template>
```

可加兜底图：

```vue
<img :src="dynamicImg" @error="e => e.target.src = '/images/default.png'" />
```
## 优点 ：
因为它把“图片资源管理”从打包阶段移出了首屏关键路径，核心是这几点：

JS 包更小
assets 里被代码引用的图，会进入构建分析和资源映射；图很多时会增加构建产物里的映射信息。放 public/CDN 后，前端只是一条字符串 URL，不需要把这些图纳入模块图。

首屏请求更可控
只有真正出现在首屏 DOM 里的图片才会发请求。不会因为你写了 glob(eager)、静态 import 或其他预加载策略把大量非首屏图提前拉下来。

浏览器并发带宽留给关键资源
首屏阶段主要竞争的是 HTML/CSS/JS + 首屏关键图。减少非必要图片参与后，关键资源更快到达，FCP/LCP 通常更好。

CDN 缓存与分发更优
大图走 CDN（就近、长缓存）比跟应用包一起走更高效，用户二次访问也更容易命中缓存。

一句话：public/CDN + 动态 URL 的模式让“非首屏图片按需加载”，避免它们在首屏阶段占用包体和网络预算，所以首屏更轻。

## 缺点：
如果图片在 src/assets 且被 Vite 正常处理，构建后会自动带 hash 文件名，通常不需要时间戳。
如果是 public/CDN 纯字符串 URL，Vite 不接管，你要自己加 ?v=版本号。
