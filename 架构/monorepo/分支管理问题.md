一是分支管理问题，多个applications项目如何管理分支，比如每个项目都有dev、test、master分支还是说共用同一个分支。

## 可以自己实现，用 Git 原生命令就够

本质上 `lerna changed` 就是封装了 `git diff`，完全可以自己写。

---

### 核心原理

```bash
# lerna changed 底层就是这个
git diff --name-only origin/main...HEAD
```

输出类似：
```
apps/web/src/index.ts
apps/web/package.json
packages/ui/src/button.tsx
```

---

### 自己实现的脚本

```bash
#!/bin/bash
# ci-build.sh

CHANGED=$(git diff --name-only origin/main...HEAD)

echo "=== 变动文件 ==="
echo "$CHANGED"

# 检测哪些包有变动
check_changed() {
  local pkg=$1
  echo "$CHANGED" | grep -q "^$pkg/" && return 0 || return 1
}

# packages/ui 变动 → 所有依赖它的 app 都要重建
UI_CHANGED=false
if check_changed "packages/ui"; then
  echo "📦 packages/ui 有变动，开始构建..."
  cd packages/ui && npm run build && cd -
  UI_CHANGED=true
fi

# apps/web 变动 OR ui 变动 → 重建 web
if check_changed "apps/web" || [ "$UI_CHANGED" = true ]; then
  echo "🌐 apps/web 开始构建..."
  cd apps/web && npm run build && cd -
  echo "🚀 apps/web 部署..."
  # scp / rsync / docker build 等部署命令
fi

# apps/admin 变动 OR ui 变动 → 重建 admin
if check_changed "apps/admin" || [ "$UI_CHANGED" = true ]; then
  echo "🔧 apps/admin 开始构建..."
  cd apps/admin && npm run build && cd -
fi

echo "=== 构建完成 ==="
```

### 挂到 Git Hook 上

```bash
# /path/to/repo.git/hooks/post-receive
#!/bin/bash

# 切换到工作目录
cd /var/www/myproject

# 拉最新代码
git pull origin main

# 执行增量构建脚本
bash ./ci-build.sh
```

---

### 效果对比

```
push 代码
    ↓
git diff 检测变动
    ↓
packages/ui 变动
    ├─→ 构建 packages/ui
    ├─→ 构建 apps/web（依赖 ui）
    └─→ 构建 apps/admin（依赖 ui）

apps/web 单独变动
    └─→ 只构建 apps/web，其余跳过
```

## lerna 方案

# 只对上次 commit 之后有变动的包执行命令
lerna run build --since HEAD~1
# 只对比 main 分支有变动的包执行
lerna run build --since origin/main
# 只对有变动的包跑测试
lerna run test --since origin/main
# 查看哪些包有变动
lerna changed
```
```
效果：
push 到 dev 分支
    ↓
lerna changed 检测
    ↓
apps/web  有变动 → 执行 build + deploy
apps/admin 无变动 → 跳过
packages/ui 有变动 → 执行 build，触发依赖它的 app 重新构建  这个方法可以自己实现不

### 和 lerna 方案对比

| | 自写脚本 | lerna --since |
|---|---|---|
| 依赖工具 | 零依赖，只用 git | 需要 lerna |
| 依赖链感知 | 需要手动维护 | 自动读 package.json |
| 灵活性 | ✅ 完全自定义 | 受限于 lerna 配置 |
| 维护成本 | 包多了脚本会变复杂 | 低 |

**包少（< 10个）** 自己写脚本完全够用，包多了依赖关系复杂再考虑引入 lerna / turborepo。
