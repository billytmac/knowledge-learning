```jsonc
{
  "extends": "@workspace/typescript-config/react-library.json",
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@workspace/ui/*": ["./src/*"]
    }
  },
  "include": ["."],
  "exclude": ["node_modules", "dist"]
}

```
逐行解释：

**`extends`** — 继承公共 TS 配置，不用每个包重复写一堆配置项。

**`compilerOptions`**
- `baseUrl: "."` — 设置路径解析的根目录为当前目录
- `paths` — 路径别名，把 `@workspace/ui/*` 映射到 `./src/*`，这样导入时可以写：
  ```ts
  import { Button } from '@workspace/ui/button'
  // 实际解析为 ./src/button
  ```

**`include: ["."]`** — 编译当前目录下所有 TS 文件。

**`exclude`** — 排除 `node_modules` 和 `dist`，不参与编译。

---

问题的根源是配置冲突——base.json 设了 "moduleResolution": "NodeNext"（要求显式 .js 后缀），但 Next.js 的打包器无法将 .js 解析到 .ts 源文件。解决方法是在 react-library.json 中覆盖为 "module": "ESNext" + "moduleResolution": "Bundler"，与 nextjs.json 保持一致，因为 packages/lib 最终是被 Next.js 打包消费的，而非直接运行在 Node.js 中。这样就可以不带 .js 后缀导入，两边都不报错。
