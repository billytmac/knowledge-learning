# Record<K, V> 是 TypeScript 内置的工具类型，表示键为 K 类型、值为 V 类型的对象。

## 举例：

    ``` ts
    Record<string, PickerDataItem>
    // 等价于
    {
     [key: string]: PickerDataItem
    }
    interface PickerDataItem {
        selectedData: number[]
        dataList: { label: string; value: number }[][]
        extendSelectedData: Record<string, unknown>
        // 用 unknown 表示扩展数据结构不确定，比 any 更安全。
    }
    // 实际数据长这样
    {
    "role1": { selectedData: [], dataList: [], extendSelectedData: {} },
    "role2": { selectedData: [], dataList: [], extendSelectedData: {} },
       ....
    }

# 不是说构造函数就是实例类型，是 TypeScript 里同一个名字 LoginService 在不同位置有不同含义：
在运行时（值的位置），LoginService 是构造函数：
const instance = new LoginService(...)  // 这里 LoginService 是构造函数（值）
在类型位置，LoginService 自动代表它的实例类型：
let x: LoginService  // 这里 LoginService 是实例类型，x 拥有实例的属性和方法
这是 TypeScript 的设计——声明一个 class 时，它同时引入了两个东西：
写法	含义
x: LoginService	实例类型（有 isLoadSDK、方法等）
x: typeof LoginService	类本身的类型（有 new(...) 构造签名）
所以 loginInstance: LoginService 写在类型注解位置，表达的就是"一个 LoginService 的实例"，完全正确。
